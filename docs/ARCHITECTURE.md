# 转刀刀游戏架构文档

## 系统架构概述

转刀刀游戏采用模块化架构设计，各模块职责明确，便于维护和扩展。整体架构基于经典的MVC（Model-View-Controller）模式，但针对游戏特性进行了优化。

## 核心架构图

```
┌─────────────────────────────────────────────────────────────┐
│                        Game (Controller)                     │
├─────────────────────────────────────────────────────────────┤
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────────┐  │
│  │   Player    │  │     NPC     │  │  CombatSystem       │  │
│  │   (Model)   │  │   (Model)   │  │   (Controller)      │  │
│  └─────────────┘  └─────────────┘  └─────────────────────┘  │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────────┐  │
│  │ MapGenerator│  │ Optimization│  │     UIManager       │  │

│  └─────────────┘  └─────────────┘  └─────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

## 核心模块详解

### 1. Game类 (游戏控制器)

**职责**:
- 游戏主循环管理
- 实体生命周期管理
- 游戏状态维护
- 输入事件处理

**关键方法**:
- `start()`: 启动游戏循环
- `stop()`: 停止游戏
- `update()`: 更新游戏状态
- `render()`: 渲染游戏画面

**依赖关系**:
- 所有其他模块

### 2. Player类 (玩家模型)

**职责**:
- 玩家状态管理（位置、生命值、刀数量）
- 移动逻辑处理
- 攻击力计算

**关键属性**:
- `position`: 当前位置
- `knives`: 刀收集状态
- `isAlive`: 存活状态

### 3. NPC类 (NPC模型)

**职责**:
- NPC行为逻辑
- 巡逻和攻击决策
- 刀掉落管理

**关键行为**:
- 随机巡逻
- 玩家接近检测
- 攻击决策

### 4. MapGenerator类 (地图模型)

**职责**:
- 随机地图生成
- 障碍物放置
- 可通行区域验证

**算法特点**:
- 边界固定为障碍物
- 内部随机生成障碍物
- 确保至少一个可通行区域

### 5. CombatSystem类 (战斗控制器)

**职责**:
- 战斗逻辑处理
- 攻击力计算
- 战斗结果判定

**战斗公式**:
```
攻击力 = 红刀数量×4 + 黄刀数量×2 + 蓝刀数量×1
```

### 6. OptimizationManager类 (性能管理器)

**职责**:
- 性能监控和优化
- 动态调整游戏参数
- 内存管理

**优化策略**:
-H 脏矩形渲染
-H 空间分割碰撞检测
-H 对象池技术
-H 动态NPC数量调整

### 7. UIManager类 (视图管理器)

**职责**:
- 游戏界面渲染
- 状态信息显示
- 用户交互反馈

## 数据流设计

### 游戏循环数据流

```
输入事件 → Game.update() → 实体状态更新 → 碰撞检测 → 战斗处理 → Game.render() → 画面渲染
```

### 战斗系统数据流

```
玩家/NPC位置 → 碰撞检测 → 攻击力计算 → 战斗结果 → 状态更新 → UI反馈
```

## 性能优化架构

### 1. 渲染优化
- **脏矩形技术**: 只重绘发生变化的部分
- **批量渲染**: 同类实体批量绘制
- **离屏Canvas**: 预渲染静态元素

### 2. 计算优化
- **空间分割**: 使用网格系统优化碰撞检测
- **对象池**: 复用实体对象减少GC压力
- **LOD技术**: 根据距离简化计算

### 3. 内存优化
- **实体复用**: NPC和刀对象的循环使用
- **资源预加载**: 图片和声音资源预加载
- **及时清理**: 无用对象的及时释放
