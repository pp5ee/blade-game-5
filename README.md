# 转刀刀 - Knife Throwing Game

一个基于HTML5 Canvas的单机网页游戏，玩家控制角色在随机生成的地图上收集不同颜色的刀，与主动攻击的NPC进行战斗。

## 游戏特色

- 🎮 **单机网页游戏** - 无需服务器，直接在浏览器中运行
- 🗺️ **随机地图生成** - 每次游戏都有不同的地图布局
- 🔪 **三种颜色刀系统** - 红、黄、蓝三种刀，伤害梯度为2倍差距
- 🤖 **主动攻击NPC** - NPC会主动寻找并攻击玩家
- ⚔️ **基于数量的战斗系统** - 战斗结果取决于刀的数量和颜色
- 🎨 **像素风格UI** - 复古像素艺术风格界面

## 游戏玩法

1. **移动角色** - 使用方向键或WASD控制角色移动
2. **收集刀** - 移动到刀的位置自动收集
3. **战斗NPC** - 与NPC相遇时自动战斗
4. **获得战利品** - 击败NPC获得对方收集的刀
5. **游戏目标** - 击败所有NPC获得胜利

## 伤害系统

- 🔴 **红色刀** = 黄色刀 × 2 = 蓝色刀 × 4
- 战斗时根据刀的数量和颜色计算总攻击力
- 攻击力高的角色在战斗中获胜

## 技术架构

- **前端**: HTML5 Canvas + 纯JavaScript
- **游戏循环**: requestAnimationFrame
- **地图系统**: 基于网格的随机生成算法
- **实体系统**: 模块化的玩家和NPC行为管理
- **战斗系统**: 基于伤害计算的战斗解析

## 快速开始

### 前置要求

- 现代浏览器（Chrome、Firefox、Safari、Edge等）
- 支持HTML5 Canvas的浏览器环境

### 安装和运行

1. **下载项目**
   ```bash
   git clone <repository-url>
   cd knife-throwing-game
   ```

2. **直接运行**
   - 使用任何HTTP服务器打开 `index.html`
   - 或直接在浏览器中打开 `index.html`（部分功能可能需要HTTP服务器）

3. **使用本地服务器运行（推荐）**
   ```bash
   # 使用Python3
   python -m http.server 8000
   # 或使用Node.js http-server
   npx http-server
   ```
   然后在浏览器中访问 `http://localhost:8000`

## 游戏控制

- **移动**: 方向键 ↑↓←→ 或 WASD
- **开始游戏**: 点击"开始游戏"按钮
- **重新开始**: 游戏结束后点击"重新开始"按钮

## 配置选项

游戏提供以下可配置参数（在 `src/game/Game.js` 中）：

- `mapSize`: 地图尺寸（默认20x15网格）
- `cellSize`: 每个网格的像素大小（默认40px）
- `maxNPCs`: 最大NPC数量（默认5个）
- `maxKnives`: 最大刀数量（默认15把）
- `obstacleDensity`: 障碍物密度（默认20%）

## 性能优化

游戏已实施以下性能优化措施：

### 渲染优化
- **帧率控制**: 稳定60FPS游戏循环
- **视锥体剔除**: 只渲染可见区域的游戏对象
- **性能监控**: 实时FPS和实体数量显示

### 碰撞检测优化
- **距离阈值**: 只检查相邻实体的碰撞
- **空间优化**: 减少不必要的碰撞计算
- **缓存机制**: 优化重复计算性能

### 内存管理
- **及时清理**: 自动移除死亡NPC和已收集的刀
- **事件管理**: 正确绑定和解绑事件监听器
- **资源优化**: 高效的内存使用模式

## 游戏平衡性

### 伤害系统
- **红色刀**: 4倍伤害（基础伤害 × 4）
- **黄色刀**: 2倍伤害（基础伤害 × 2）
- **蓝色刀**: 1倍伤害（基础伤害 × 1）

### NPC行为
- **移动冷却**: 500ms移动间隔
- **攻击范围**: 5格距离内主动攻击
- **智能寻路**: 根据玩家位置智能移动

### 难度设计
- **渐进难度**: NPC数量和强度动态调整
- **平衡资源**: 刀生成数量和分布优化
- **公平战斗**: 基于数量的战斗系统确保公平性
